# Slater Reread Pass

## Source Pressure

Slater's stable property cluster view is not a bare recurrence view. The key distinction is between instance stability and cliquish stability. Instance stability asks whether a particular item keeps the cluster once it has it. Cliquish stability asks whether finding enough of the subcluster licenses an expectation about the larger cluster across the relevant counterfactual range.

That matters for this paper because the current section is a little too quick to say that "unexplained stability is still too thin." Slater's stability is already projectibility-shaped and domain-relative. The contrast should not be "Slater gives mere stability, then we add projectibility." The contrast should be: Slater gives the right first projectibility demand, and the diagnostic then asks which stronger securing claim, if any, has also been exhibited.
